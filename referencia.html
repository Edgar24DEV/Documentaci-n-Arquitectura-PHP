Roadmap del
Arquitecto PHP

12 meses para pasar de Laravel developer a Software Architect.

Plan personal
Mes 1 — 2
Clean Code en cada PR
Nombres claros, funciones pequeñas, sin comentarios innecesarios. Configura PHPStan nivel 6+ en tu proyecto. Lee "Clean Code" de Uncle Bob.
Mes 3 — 4
SOLID + primeros tests
Refactoriza Fat Controllers. Introduce Form Requests, Actions y el primer Repository con interfaz. Empieza a escribir tests con InMemoryRepositories usando Pest.
Mes 5 — 6
Arquitectura Hexagonal + pirámide de tests
Construye una API REST con la estructura de Bounded Contexts. Tests unitarios de dominio y use cases sin framework. Lee "Laravel Beyond CRUD" de Brent Roose.
Mes 7 — 8
DDD + Value Objects en Commands/Queries
Modela Aggregates con invariantes, Domain Events y Value Objects. Aplica VOs en Commands y Queries. Object Mothers para tests. Lee "Implementing DDD" de Vaughn Vernon.
Mes 9 — 10
CQRS + Event Bus
Finders, Queriers y AuthorizationServices. Procesa Commands en background con Laravel Horizon + Redis. Integration tests del Querier.
Mes 11 — 12
Microservicios y Event Sourcing
Cada Bounded Context puede ser un microservicio independiente. Explora Event Sourcing, API Gateway y Circuit Breaker. Lee "Building Microservices" de Sam Newman.
