Exception Hierarchy — 3 capas 
// ═══ DOMAIN LAYER ═══
namespace App\DDD\Shared\Domain\Exception;

abstract class DomainException extends \DomainException {}

final class EntityNotFoundException extends DomainException
{
    public function __construct(
        public readonly string $entity,
        public readonly string $id,
    ) {
        parent::__construct("{$entity} not found: {$id}");
    }
}

final class BusinessRuleViolationException extends DomainException
{
    public function __construct(
        string $rule,
        public readonly array $context = [],
    ) {
        parent::__construct($rule);
    }
}

final class InvalidValueObjectException extends DomainException {}

// ═══ APPLICATION LAYER ═══
namespace App\DDD\Shared\Application\Exception;

abstract class ApplicationException extends \Exception {}
final class UnauthorizedException extends ApplicationException {}
final class CommandValidationException extends ApplicationException {}

// ═══ INFRASTRUCTURE LAYER ═══
namespace App\DDD\Shared\Infrastructure\Exception;

abstract class InfrastructureException extends \Exception {}
final class PersistenceException extends InfrastructureException {}
final class ExternalServiceException extends InfrastructureException {}
app/Exceptions/Handler.php — Mapeo HTTP 
public function register(): void
{
    $this->renderable(function (EntityNotFoundException $e) {
        return response()->json(['error' => $e->getMessage()], 404);
    });

    $this->renderable(function (UnauthorizedException $e) {
        return response()->json(['error' => 'Forbidden'], 403);
    });

    $this->renderable(function (BusinessRuleViolationException $e) {
        return response()->json([
            'error'   => $e->getMessage(),
            'context' => $e->context,
        ], 422);
    });
}

⚠️
Reglas críticas:
• Nunca $guarded = [] — siempre define $fillable
• Nunca env() en código — usa config()
• Nunca SQL raw con interpolación — usa bindings
• Nunca hardcodear secrets en código

Anti-Patterns
a Evitar

Errores comunes que rompen la arquitectura.

Referencia
Anti-Pattern Problema Solución
Anemic Domain Model Entities son solo getters/setters sin lógica Comportamiento e invariantes dentro de las entidades
Smart UI Lógica de negocio en Controllers o Views Thin Controllers → Application → Domain
Leaky Abstraction Domain extiende Eloquent Model Entidades puras; Mapper en Infrastructure
Service Locator app() / resolve() en lógica Constructor injection únicamente
Primitive Obsession Strings/ints donde deberían ser VOs Modelar conceptos de dominio como Value Objects
Fat Repository Lógica de negocio en repositorios Repos solo persisten; Domain Services deciden
God Command Un Command hace 5 cosas diferentes Un Command = una operación atómica
Mixed CQRS Query Handler modifica estado Separación estricta: queries son read-only

Tu camino

Roadmap del
Arquitecto PHP

12 meses para pasar de Laravel developer a Software Architect.

Plan personal
Mes 1 — 2
Clean Code en cada PR
Nombres claros, funciones pequeñas, sin comentarios innecesarios. Configura PHPStan nivel 6+ en tu proyecto. Lee "Clean Code" de Uncle Bob.
Mes 3 — 4
SOLID + primeros tests
Refactoriza Fat Controllers. Introduce Form Requests, Actions y el primer Repository con interfaz. Empieza a escribir tests con InMemoryRepositories usando Pest.
Mes 5 — 6
Arquitectura Hexagonal + pirámide de tests
Construye una API REST con la estructura de Bounded Contexts. Tests unitarios de dominio y use cases sin framework. Lee "Laravel Beyond CRUD" de Brent Roose.
Mes 7 — 8
DDD + Value Objects en Commands/Queries
Modela Aggregates con invariantes, Domain Events y Value Objects. Aplica VOs en Commands y Queries. Object Mothers para tests. Lee "Implementing DDD" de Vaughn Vernon.
Mes 9 — 10
CQRS + Event Bus
Finders, Queriers y AuthorizationServices. Procesa Commands en background con Laravel Horizon + Redis. Integration tests del Querier.
Mes 11 — 12
Microservicios y Event Sourcing
Cada Bounded Context puede ser un microservicio independiente. Explora Event Sourcing, API Gateway y Circuit Breaker. Lee "Building Microservices" de Sam Newman.
