Value Objects con ::create() estático

Los VOs nunca se instancian con new desde fuera. El constructor es privado y la creación pasa siempre por el método estático ::create(), que centraliza la validación. Así es imposible construir un VO inválido.

Shared/Domain/ValueObjects/Money.php 
final class Money
{
    // Constructor privado: nadie puede hacer new Money()
    private function __construct(
        private readonly int    $cents,
        private readonly string $currency,
    ) {}

    public function dddGetCents(): int { return $this->cents; }
    public function dddGetCurrency(): string { return $this->currency; }

    // Única puerta de entrada — valida antes de construir
    public static function create(int $cents, string $currency): self
    {
        if ($cents < 0)
            throw new InvalidArgumentException('El importe no puede ser negativo');
        if (!preg_match('/^[A-Z]{3}$/', $currency))
            throw new InvalidArgumentException("Moneda inválida: {$currency}");

        return new self($cents, $currency);
    }

    public static function zero(string $currency): self
    {
        return self::create(0, $currency);
    }

    public function add(Money $other): self
    {
        if ($this->currency !== $other->dddGetCurrency())
            throw new InvalidArgumentException('No se pueden sumar monedas distintas');

        return self::create($this->cents + $other->dddGetCents(), $this->currency);
    }

    public function applyDiscount(float $rate): self
    {
        return self::create(intval($this->cents * (1 - $rate)), $this->currency);
    }
}

// Todos los VOs siguen el mismo patrón:
final class CustomerId
{
    private function __construct(public readonly string $value) {}

    public static function create(string $value): self
    {
        if (!Uuid::isValid($value))
            throw new InvalidArgumentException("CustomerId inválido: {$value}");
        return new self($value);
    }

    public static function generate(): self
    {
        return new self(Str::uuid());
    }
}

Command con primitivos — los VOs se crean dentro del Command

💡
El Command recibe primitivos pero expone VOs. El constructor acepta tipos simples (reutilizable desde cualquier contexto), pero los convierte a VOs internamente. El Handler accede a $cmd->customerId() y ya recibe un CustomerId — sin conversiones en ningún sitio externo.
Order/Application/PlaceOrder/PlaceOrderCommand.php 
// ✅ Recibe primitivos → expone VOs — reutilizable desde cualquier capa
final class PlaceOrderCommand
{
    private function __construct(
        private readonly CustomerId    $customerId,
        private readonly OrderItemList $items,
        private readonly Currency      $currency,
    ) {}

    public static function create(string $customerId, array $items, string $currency): self
    {
        return new self(
            CustomerId::create($customerId),
            OrderItemList::create($items),
            Currency::create($currency)
        );
    }

    public function dddGetCustomerId(): CustomerId    { return $this->customerId; }
    public function dddGetItems():      OrderItemList { return $this->items; }
    public function dddGetCurrency():   Currency      { return $this->currency; }
}

// Controller: pasa strings — el Command se encarga de la conversión
class OrderController extends Controller
{
    public function store(StoreOrderRequest $request, PlaceOrderHandler $handler)
    {
        $orderId = $handler->handle(PlaceOrderCommand::create(
            customerId: auth()->id(),
            items:      $request->validated('items'),
            currency:   $request->validated('currency', 'EUR'),
        ));
        return response()->json(['id' => $orderId->value()], 201);
    }
}

Handler sin persistencia — Creator como Domain Service

💡
Creator en Domain, no en Application. El OrderCreator es un Domain Service: solo construye el Aggregate con reglas puras, sin repositorios. El Handler orquesta: llama a $cmd->dddGetCustomerId() (ya es un VO, construido en el Command), pasa los VOs al Creator, y persiste el resultado. Ninguna capa convierte primitivos a VOs excepto el propio Command.
Order/Domain/Services/OrderCreator.php — Domain Service (puro) 
// ✅ Domain Service — sin repositorios, sin infraestructura
// Solo conoce las reglas de negocio de "construir un pedido válido"
final class OrderCreator
{
    public function __construct(
        private readonly ProductRepository   $products,   // port, no Eloquent
        private readonly OrderPricingService $pricing,    // otro Domain Service
    ) {}

    // Devuelve el Aggregate construido — NO persiste
    public function create(CustomerId $customerId, OrderItemList $items, Currency $currency): Order
    {
        $order = Order::create($customerId, $currency);

        foreach ($items->all() as $item) {
            // ProductRepository es una interfaz — el Creator no sabe si es Eloquent o InMemory
            $product = $this->products->findById($item->dddGetProductId())
                ?? throw new ProductNotFound($item->dddGetProductId());

            $price = $this->pricing->priceFor($product, $item->dddGetQuantity());
            $order->addItem($product, $item->dddGetQuantity(), $price);
        }

        return $order;  // ← devuelve el objeto, no persiste
    }
}
Order/Application/PlaceOrder/PlaceOrderHandler.php 
// Handler: construye VOs → llama al Creator (Domain) → persiste (Application)
final class PlaceOrderHandler
{
    public function __construct(
        private readonly OrderCreator    $creator,  // Domain Service
        private readonly OrderRepository $orders,   // Output port
        private readonly EventDispatcher $events,   // Output port
    ) {}

    public function handle(PlaceOrderCommand $cmd): OrderId
    {
        // Los VOs ya vienen construidos desde el Command — sin conversión aquí
        $order = $this->creator->create(
            $cmd->dddGetCustomerId(),  // CustomerId VO
            $cmd->dddGetItems(),       // OrderItemList VO
            $cmd->dddGetCurrency(),    // Currency VO
        );

        $this->orders->save($order);
        array_map(fn($e) => $this->events->dispatch($e), $order->pullEvents());

        return $order->id;
    }
}

final class ConfirmOrderCommand
{
    private OrderId $orderId;

    public function __construct(string $orderId)
    {
        $this->orderId = OrderId::create($orderId);  // VO construido aquí
    }

    public function orderId(): OrderId { return $this->orderId; }
}

final class ConfirmOrderHandler
{
    public function __construct(
        private readonly OrderRepository $orders,
        private readonly EventDispatcher $events,
    ) {}

    public function handle(ConfirmOrderCommand $cmd): void
    {
        // $cmd->orderId() ya ES un OrderId — el Command lo construyó
        $order = $this->orders->findById($cmd->orderId())
            ?? throw new OrderNotFound($cmd->orderId());

        $order->confirm();

        $this->orders->save($order);
        array_map(fn($e) => $this->events->dispatch($e), $order->pullEvents());
    }
}
✅
La clave: OrderCreator (Domain Service) no sabe nada de persistencia — devuelve el Aggregate. El Handler persiste el resultado. Los VOs se construyen en el Command/Query — el Handler los consume ya validados. Ninguna capa externa hace ::create() excepto el propio Command o Query.

Query con primitivos — mismo patrón

⚠️
Patrón idéntico en el Read Side. La Query recibe primitivos y expone VOs mediante métodos. El Handler llama a $q->dddGetOrderId() y ya recibe un OrderId — sin construcciones en el Handler.
Order/Application/Find/GetOrderQuery.php + GetOrderHandler.php 
// Query: recibe primitivos, expone VOs — reutilizable desde cualquier contexto
final class GetOrderQuery
{
    private function __construct(
        private readonly OrderId $orderId,
        private readonly UserId  $requesterId,
    ) {}

    public static function create(string $orderId, string $requesterId): self
    {
        return new self(
            OrderId::create($orderId),
            UserId::create($requesterId)
        );
    }

    public function dddGetOrderId():     OrderId { return $this->orderId; }
    public function dddGetRequesterId(): UserId   { return $this->requesterId; }
}

final class GetOrderHandler
{
    public function __construct(
        private readonly OrderFinder               $finder,
        private readonly OrderQuerier              $querier,
        private readonly OrderAuthorizationService $authz,
    ) {}

    public function handle(GetOrderQuery $q): OrderSummaryDTO
{
    // VOs vienen de la Query — sin conversión en el Handler
    $order = $this->finder->findOrFail($q->dddGetOrderId());
    $this->authz->ensureCanView($q->dddGetRequesterId(), $order);
    return $this->querier->getSummary($q->dddGetOrderId());
}
}

// Querier: interface en Application, implementación en Infra
interface OrderQuerier
{
    public function getSummary(OrderId $id): OrderSummaryDTO;
    /** @return OrderSummaryDTO[] */
    public function findByCustomer(CustomerId $id): array;
}

final class PlaceOrderHandler
{
    public function handle(PlaceOrderCommand $cmd): array
    {
        // Construye el pedido
        $order = Order::create(
            new CustomerId($cmd->dddGetCustomerId()),  // (1) Viola regla VO
            new Currency($cmd->dddGetCurrency()),     // (1) Viola regla VO
        );

        foreach ($cmd->dddGetItems() as $item) {
            $product = DB::table('products')         // (2) Viola DIP
                ->where('id', $item->dddGetProductId())
                ->first();
            $order->addItem($product, $item->dddGetQuantity());
        }

        // Persiste directamente
        OrderModel::create([                          // (3) Viola DDD/Application
            'customer_id' => $cmd->dddGetCustomerId()->value(),
            'total'       => $order->total()->dddGetCents(),
        ]);

        // Envía email
        Mail::to($cmd->dddGetEmail())->send(OrderPlaced::create());  // (4) Acoplo Infra

        return ['id' => $order->id->value()];           // (5) Viola CQRS/VOs
    }
}
