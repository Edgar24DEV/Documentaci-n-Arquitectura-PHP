Testing de
Arquitectura

Una buena arquitectura es aquella que se puede testear sin base de datos, sin HTTP y sin efectos secundarios. Los tests son el mayor indicador de calidad de diseño.

20 puntos

La arquitectura hexagonal paga su precio al escribir tests: el dominio no tiene dependencias de Laravel, por tanto los tests de Use Cases son tests unitarios puros, sin base de datos, sin HTTP, sin colas. Simplemente PHP.

Si un test de un Use Case necesita levantar el framework o la BD, es una señal de que la arquitectura está rota.

Unit Test — dominio puro (sin framework)

Los tests de dominio son los más valiosos. Prueban las invariantes del negocio directamente, en microsegundos, sin ninguna dependencia externa.

tests/Unit/Domain/OrderTest.php — Pest 
it('accumulates items and calculates total correctly', function () {
    $order = Order::create(
        CustomerId::create('customer-1'),
        Currency::create('EUR'),
    );

    $product = ProductMother::withPrice(Money::of(1000, 'EUR'));  // 10€
    $order->addItem($product, Quantity::create(3));

    expect($order->total()->dddGetCents())->toBe(3000)
         ->and($order->total()->dddGetCurrency())->toBe('EUR');
});

it('throws when adding item to a confirmed order', function () {
    $order = OrderMother::confirmedWithOneItem();

    expect(fn() => $order->addItem(ProductMother::random(), Quantity::create(1)))
        ->toThrow(OrderAlreadyConfirmed::class);
});

it('records an OrderConfirmed event on confirm', function () {
    $order = OrderMother::pendingWithItems();
    $order->confirm();

    $events = $order->pullEvents();
    expect($events)->toHaveCount(2)  // OrderCreated + OrderConfirmed
         ->and($events[1])->toBeInstanceOf(OrderConfirmed::class);
});

Unit Test — Use Case con InMemory Repository

Los tests de Use Cases usan implementaciones in-memory de los repositorios. Sin base de datos real, sin Laravel, ejecución en <1ms. Esto solo es posible gracias a DIP.

tests/Unit/Application/PlaceOrderHandlerTest.php 
use Tests\Doubles\InMemoryOrderRepository;
use Tests\Doubles\InMemoryProductRepository;
use Tests\Doubles\SpyEventDispatcher;

it('places an order and dispatches domain events', function () {
    // Arrange — sin BD, sin framework
    $orders   = new InMemoryOrderRepository();
    $products = new InMemoryProductRepository();
    $events   = new SpyEventDispatcher();
    $product  = ProductMother::withPrice(Money::of(2000, 'EUR'));
    $products->save($product);

    $handler = new PlaceOrderHandler($orders, $products, $events);
    $command = PlaceOrderCommand::create(
        CustomerId::create('customer-1'),
        OrderItemList::fromArray([[
            'productId' => $product->id->value(),
            'quantity'  => 2,
        ]]),
        Currency::create('EUR'),
    );

    // Act
    $orderId = $handler->handle($command);

    // Assert
    $saved = $orders->findById($orderId);
    expect($saved)->not()->toBeNull()
         ->and($saved->id)->toEqual($orderId)
         ->and($saved->total()->dddGetCents())->toBe(4000);

    expect($events->dispatched())->toHaveCount(1)
         ->and($events->dispatched()[0])->toBeInstanceOf(OrderCreated::class);
});

it('throws when a product is not found', function () {
    $handler = new PlaceOrderHandler(
        new InMemoryOrderRepository(),
        new InMemoryProductRepository(),  // vacío
        new SpyEventDispatcher(),
    );

    expect(fn() => $handler->handle(PlaceOrderCommandMother::random()))
        ->toThrow(ProductNotFound::class);
});

Test Doubles — InMemory Repository

Los repositorios in-memory son implementaciones del interface de dominio que guardan datos en un array. Son la clave para tests de use cases rápidos y sin dependencias.

tests/Doubles/InMemoryOrderRepository.php 
final class InMemoryOrderRepository implements OrderRepository
{
    /** @var Order[] */
    private array $store = [];

    public function findById(OrderId $id): ?Order
    {
        return $this->store[$id->value()] ?? null;
    }

    public function save(Order $order): void
    {
        $this->store[$order->id->value()] = $order;
    }

    /** Helpers para asserts en tests */
    public function count(): int
    {
        return count($this->store);
    }

    public function all(): array
    {
        return array_values($this->store);
    }
}

// SpyEventDispatcher: registra los eventos despachados
final class SpyEventDispatcher implements EventDispatcher
{
    private array $dispatched = [];

    public function dispatch(object $event): void
    {
        $this->dispatched[] = $event;
    }

    public function dispatched(): array
    {
        return $this->dispatched;
    }
}

Integration Test — con base de datos real

✅
Cuándo usar integration tests: para probar que el Mapper, el Repository Eloquent y la BD real funcionan juntos. No para la lógica de negocio — eso ya lo cubren los unit tests.
tests/Integration/EloquentOrderRepositoryTest.php 
use Illuminate\Foundation\Testing\RefreshDatabase;

uses(RefreshDatabase::class);

it('persists and retrieves an order by id', function () {
    $repo  = new EloquentOrderRepository();
    $order = OrderMother::confirmed();

    $repo->save($order);

    $found = $repo->findById($order->id);

    expect($found)->not()->toBeNull()
         ->and($found->id)->toEqual($order->id)
         ->and($found->total()->dddGetCents())->toBe($order->total()->dddGetCents());
});

it('returns null for a non-existent order', function () {
    $repo = new EloquentOrderRepository();
    expect($repo->findById(OrderId::create('non-existent')))->toBeNull();
});

Object Mothers — datos de test legibles

Los Object Mothers son factories de objetos de dominio para tests. Centralizan la creación de objetos complejos y hacen los tests mucho más legibles.

tests/Mothers/OrderMother.php 
final class OrderMother
{
    public static function pending(): Order
    {
        return Order::create(
            CustomerId::create(Str::uuid()),
            Currency::create('EUR'),
        );
    }

    public static function pendingWithItems(int $count = 2): Order
    {
        $order = self::pending();
        for ($i = 0; $i < $count; $i++) {
            $order->addItem(ProductMother::random(), Quantity::create(1));
        }
        return $order;
    }

    public static function confirmed(): Order
    {
        $order = self::pendingWithItems();
        $order->confirm();
        return $order;
    }
}
