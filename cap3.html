Error común: estructurar por capa técnica primero (Domain/, Application/…). En DDD el Bounded Context es el ciudadano de primera clase.
Estructura DDD — Bounded Context primero 
src/
├── Order/                       ◀ Bounded Context
│   ├── Domain/                  Cero deps de Laravel
│   │   ├── Order.php            Aggregate Root
│   │   ├── OrderId.php          Value Object
│   │   ├── OrderStatus.php      Enum de dominio
│   │   ├── Events/              OrderPlaced, OrderConfirmed
│   │   ├── Exceptions/          OrderNotFound
│   │   ├── Repositories/
│   │   │   └── OrderRepositoryInterface.php Interface (Puerto)
│   │   └── Services/
│   │       └── OrderPricingService.php  Domain Service
│   ├── Application/
│   │   ├── PlaceOrder/
│   │   │   ├── PlaceOrderCommand.php    con VOs
│   │   │   └── PlaceOrderHandler.php
│   │   └── Find/
│   │       ├── OrderFinder.php          usa el Repo
│   │       ├── OrderQuerierInterface.php  Interface proyecciones
│   │       ├── GetOrderQuery.php         con VOs
│   │       └── GetOrderHandler.php      permisos→finder→querier

│       ├── EloquentOrderRepository.php
