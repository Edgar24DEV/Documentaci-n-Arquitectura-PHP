El código limpio no es una cuestión estética. Es la diferencia entre un sistema que puedes cambiar con confianza y uno que tienes miedo de tocar.

Robert C. Martin define el buen código como aquel que "hace una cosa bien y es fácil de cambiar". Los principios no son reglas arbitrarias — son respuestas a problemas reales que todo equipo encuentra cuando un proyecto crece.

Funciones pequeñas, un nivel de abstracción

💡
Naming en DDD/PHP 8.3: los Value Objects usan dddGet*() como prefijo de getters para claridad en contexto DDD. Los guards de invariantes usan guard*(). Los métodos de Domain Events usan eventName(). Las interfaces llevan el sufijo Interface. Consistencia total en todo el proyecto.

Una función debe hacer una cosa. Si necesitas escribir "y" para describirla, está haciendo demasiado. Todo el código dentro de una función debe estar en el mismo nivel de abstracción.

PlaceOrderAction.php 
class PlaceOrderAction
{
    public function execute(PlaceOrderDTO $dto): Order
    {
        // Mismo nivel de abstracción en todo el método
        $this->validate($dto);
        $total = $this->calculateTotal($dto->items, $dto->coupon);
        $order = $this->createOrder($dto, $total);
        $this->notifyCustomer($order);
        return $order;
    }

    private function calculateTotal(array $items, ?string $coupon): Money
    {
        $subtotal = OrderCalculator::subtotal($items);
        return CouponService::apply($subtotal, $coupon);
    }
}

UserController.php 
public function proc(Request $r)
{
    // check
    if (!$r->e || !$r->p) return false;

    $d = DB::table('users')->where('email', $r->e)->first();
    if (!$d) return false;

    if (password_verify($r->p, $d->password)) {
        session_start();
        $_SESSION['uid'] = $d->id;
        mail($r->e, 'Login', 'Has iniciado sesión');
        return true;
    }
    return false;
}
